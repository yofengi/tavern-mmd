# 混合态雷达法状态栏（MMD首选方案）

> 基于特征嗅探与响应式动态DOM的状态栏架构（改五版，by 黑洞猫）。MMD平台状态栏**首选本方案**；KV V4.0（statusbar.md）作为轻量备选保留。
> 现成可用的开源示例资产在 `../../assets/radar-examples/`，新做状态栏时**改造示例远快于从零写引擎**。

## 与 KV V4.0 的选型对比

| 维度 | 雷达法（本文档） | KV V4.0（statusbar.md） |
|---|---|---|
| 模型输出 | 纯键值对 `[键名=键值]`，零HTML | HTML属性式数据块（div+属性） |
| 视图构建 | JS引擎内存动态装配（createElement） | 正则替换出预制HTML骨架 |
| 模型格式出错时 | 单条数据"裸奔"显示，UI不崩 | 可能整块解析失败 |
| 未知新键名 | 特征嗅探自动路由归位 | 不支持，字段需预定义 |
| 防平台重绘 | 探针自检+零延迟重建 | 无防御 |
| 响应式 | 媒体查询宽窄屏自适应 | 固定布局 |
| 工程复杂度 | 高（引擎~13K字符，需AI辅助生成） | 低（三段固定模板） |
| 适用 | 正式卡、长线游玩、复杂数据 | 快速原型、极简面板 |

## 核心思想：数据与视图解耦

模型**只输出扁平键值对**（如 `[生命值=45/100]`），不写任何排版代码。前端雷达引擎捕获数据后在内存中动态拼装UI并一次性挂载。模型对前端渲染一无所知——**信息不对称原则**：状态栏规则里严禁向模型解释UI机制（"系统会自动折叠"之类），只写"什么情境输出什么键值对"。

## 五级数据分类理论（设计状态栏的第一步）

规划任何雷达法状态栏，先把所有字段按五级分类：

| 级别 | 定义 | 例子 | 生命周期处理 |
|---|---|---|---|
| 一类 全局锚点 | 维持世界观核心逻辑，每轮必在 | 主角名、主线目标、纪元背景、世界书触发词 | 固态：每轮强制生成+历史兜底 |
| 二类 高频变量 | 每轮大概率变化，驱动扮演 | 时间地点、生命值、好感度 | 固态：每轮强制生成+历史兜底 |
| 三类 后台暗流 | 仅模型可见的隐藏推理线索 | 真实好感、暗骰结果、杀意阈值 | 折叠隐藏或不渲染 |
| 四类 场景快照 | 高刺激特写，**残留会污染上下文** | 战损评估、极端情感/感官状态 | 液态：情境绑定生成，**绝不兜底**，离开场景即销毁 |
| 五类 低频冗余 | 真实感补充，极少变动 | 排班表、履历、装备说明 | 固态兜底，仅变动时输出 |

四类数据是长线游玩"模型智商下降"的元凶——高权重感官词汇残留在历史中会导致模型把日常互动过度推演。雷达法的**液态快照无兜底代谢机制**是核心护城河。

## 双轨生命周期

- **固态底座**（一/二/五类）：模型本轮没输出时，引擎向历史气泡逆向检索最近有效值兜底填充。降低模型输出压力，UI视觉常驻。
- **液态快照**（四类+部分三类）：无兜底。模型停止输出该键名，下一轮渲染时UI模块即被抹除。

特殊情形：
- **装备/技能槽位**：槽位名（如 `[主要武器=紫木法杖]`）每轮强制生成但**不兜底**（避免换装后旧装备残留UI）；装备释义（`[紫木法杖=效果说明]`）穿插正文输出一次，引擎跨轮回溯兜底并绑定为悬浮气泡——"槽位动态更新"与"效果永久记忆"解耦。
- **选项**：液态无兜底。模型截断没生成选项时UI物理缺失=显性报错信号，提醒玩家回溯重生成，而非兜底旧选项掩盖问题。
- **带计数器的键名**（如 `[事件总结N/3=...]`）：键名每轮变化，天然不适用兜底。

## 标准部署：四条正则（按优先级从上到下）

| 优先级 | 名称 | findRegex | replaceString | 作用 |
|---|---|---|---|---|
| 1 | 对抗代码 | `<代码>` | img onerror + MutationObserver（~1.9K字符） | 反平台强制染色注入，净化font标签为span.hl-dialogue，修复MMD全角引号bug。轻量场景可省 |
| 2 | 响应式CSS | `<css>` | `<style>`全套样式（~5.5K字符） | 排版/颜色/悬浮气泡/媒体查询折叠，与逻辑层彻底剥离 |
| 3 | 数据信标转换器 | `/\[([^=\]]+)=([^\]]+)\]\s*/g` | `<span style="display:none">[$1=$2]</span>` | 把键值对包进隐形标签，尾部`\s*`吞噬换行防排版空隙 |
| 4 | 雷达引擎 | `<ztl>` | img onerror 引擎主体（~13K字符） | 剪枝探针→全量历史扫描→嗅探路由→内存装配→挂载→防劫持巡检 |

模型第一句话/正文只需含触发标记：`<css><代码>`（部署样式与哨兵）+ 每轮末尾 `<ztl>` + 键值对数据块。

**完整可运行示例**见 `../../assets/radar-examples/西幻RPG-正则与第一句话.json`（4条正则+测试正文）与 `完整美化-日夜主题与雷达.json`（9条正则：雷达法+日夜双主题全局美化+侧边栏切换的集成案例）。

## MMD导入json格式（实测可导入）

```json
{
  "pageDepth": 2,
  "statusbar": "<css><代码>",
  "beginning": "第一句话正文……<ztl>\n[键名=键值]……",
  "regex_scripts": [
    {"id": -1, "scriptName": "规则名", "findRegex": "<css>", "replaceString": "……"}
  ]
}
```

- `statusbar`：状态栏触发标记位（部署类标记放这里）
- `beginning`：第一句话（含测试数据块）
- `regex_scripts`：每条仅4字段（id固定-1/scriptName/findRegex/replaceString），与本地酒馆正则json字段不同
- 限额仍是30条/1000/20000字符

## 防御体系（引擎代码必须遵守，给AI生成引擎时逐条核对）

| 威胁 | 现象 | 对抗策略 |
|---|---|---|
| 虚拟DOM劫持 | 点击气泡/编辑保存后状态栏消失或重影 | 挂载于原生信息节点内部；渲染后2.5秒内短程探针自检，被抹除即闭包缓存原地重建，5次巡检后自毁释放资源；数据哈希绑定节点ID防重影 |
| 作用域逃逸 | 编辑/预览框内堆叠全页所有历史数据 | 嗅探器录入气泡+编辑器双特征；向上溯源加层级阈值，超限强制收缩锁定父级 |
| 语法解析冲突 | 代码被截断/包上斜体/暴露乱码 | **物理级纯净编码**：逻辑内零双引号（统一单引号）；乘号改等效除法消灭星号；高危正则拆成字符串拼接后编译 |
| 强制染色注入 | 平台强插font标签污染上下文 | 独立MutationObserver哨兵：断监听→剥离标签→释放纯文本→重启监听 |
| 符号集幻觉 | 模型输出全角`／｜：`致解析断裂 | 全半角鲁棒正则：`[\/／]`、`/[|｜]/`；双指针 `indexOf(':')` + `indexOf('：')` |
| 渲染深度限制 | 超20轮历史数据失效"失忆" | 不依赖平台正则渲染结果，引擎内置原生正则直接扫描全部历史气泡 textContent |
| 极长上下文卡顿 | 千万字符级文档树正则阻塞 | 前置剪枝探针：先 indexOf('[') 粗筛，命中才进正则管道，扫描负担降90% |

## 状态栏规则Prompt写法（模型侧协议）

完整范例见 `../../assets/radar-examples/完整美化-状态栏规则.txt` 与 `西幻RPG-状态栏规则世界书.json`。骨架：

1. **全局红线**：正文完毕后另起一行输出`<ztl>`锚点；严格`[键名=键值]`格式；**严禁键值内嵌套方括号**（多项用换行/全角冒号/竖线`|`分割）；解释性快照穿插正文，严禁堆在数据块区
2. **核心区（每轮强制）**：逐字段列出键名、格式、示例（地点用`-`连字符层级、时间含时段词、数值`当前/最大`、`[回复轮次=N]`每轮+1、`[任务目标=...]`无目标输出"暂无"）
3. **按需区**：装备槽位/技能分类每轮生成；释义入手时正文穿插一次；背包用`|`分行，未装备物品键名加"背包"前缀
4. **快照区（情境绑定）**：状态成因`[生命状态=...]`、鉴定`[已鉴定-物品名=...]`、战斗`[遭遇敌人=名|HP:x/y|...]`**脱战即停**；`[事件总结N/3=...]`计数循环
5. **选项**：`[选项标题=...]`每轮锚定 + 3-4条`[选项=简介|内容]`
6. 明确告知模型可自由发散新键名（嗅探引擎会归位），但**不要**解释任何UI机制

## 制作工作流（给用户做雷达法状态栏时）

1. **需求与分类**：和用户确定状态栏字段清单 → 逐字段做五级分类（写进plan.md）
2. **基底选择**：优先从 `radar-examples/` 选近似示例改造；全新需求才从零设计HTML原型
3. **改造CSS**（优先级2正则）：换配色/布局，保留媒体查询折叠与悬浮气泡类
4. **改造引擎**（优先级4正则）：调整兜底白名单（一/二/五类键名入名单，四类排除）、嗅探特征字典（装备/技能/背包路由词）、UI面板映射。**逐条对照防御体系表核查**：零双引号、无星号、全半角兼容、剪枝探针、自毁巡检
5. **写状态栏规则**：按上节骨架定制，与引擎键名严格一致
6. **打包**：MMD导入json（pageDepth/statusbar/beginning/regex_scripts四字段）+ 状态栏规则进世界书条目；字符数核查（单条≤20000）
7. **实机测试**：第一句话贴测试数据块 → 看渲染 → 编辑/翻页测防劫持 → 窄屏测折叠

## 常见故障排查

| 症状 | 原因 | 处理 |
|---|---|---|
| 页面爆出代码乱码 | 引擎含双引号/星号触发平台解析 | 按纯净编码规范重写违规段 |
| 键值对裸奔显示在正文 | 模型格式崩坏（截断/嵌套方括号），引擎显性报错拒收 | 手改该段文本或回溯重生成——这是防御特性不是bug |
| 状态栏不出现 | 模型没输出`<ztl>`锚点 | 检查规则里是否明确要求每轮输出`<ztl>` |
| 编辑后状态栏消失 | 防劫持探针已过巡检期 | 正常，下一轮回复重新渲染；高频复现则增加巡检次数 |
| 某数据"失忆" | 该键名误入无兜底名单 / 超渲染深度且引擎未开全量扫描 | 核对兜底白名单；确认引擎用textContent全量扫描 |
| 窄屏排版挤压 | CSS缺媒体查询 | 补`@media (max-width:650px)`单列折叠规则 |
