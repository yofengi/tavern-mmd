# 美化风格系统总纲

定位：本文件是美化风格数据库的顶层协议，规定 token 契约、六维度模型、选择流程、整体性守护与项目级覆盖规则。

## 1. 统一 token 契约

本节定义一套规范变量名作为中间层，每个风格用它表达，再映射到全局美化(global-css)与状态栏(statusbar)各自的现有变量。

| 规范 token | 含义 | → global-css | → statusbar |
|---|---|---|---|
| `--bg` | 页面/面板底色 | `--lb` | `--bg` |
| `--surface` | 卡片/区块底色 | `--lc` | `--bg2` |
| `--surface-2` | 次级区块/渐变 | `--lcm` | 区块变体 |
| `--border` | 边框色 | `--lm` | `--border` |
| `--text` | 正文色 | `--lt` | `--t1` |
| `--text-2` | 次要文字 | `--lts` | `--t2` |
| `--text-3` | 弱化/标签 | （新增槽） | `--t3` |
| `--accent` | 主强调 | `--la` | `--accent` |
| `--accent-2` | 辅助强调 | `--lg` | 次accent/`--gold` |
| `--highlight` | 高亮底 | `--lh` | 高亮区 |
| `--shadow` | 阴影 | `--ls` | box-shadow |
| `--success/--warning/--danger` | 语义色 | （新增槽） | `--success/--gold/--danger` |
| `--radius` | 圆角刻度 | （新增槽） | border-radius |
| `--icon-filter` | 图标染色 | `--lif` | — |

映射只是把同一套色板填进两边现有变量，不新建渲染逻辑；global-css 缺的槽（text-3/语义色/radius）按需在项目 CSS 补，不改 global-css.md 主体。

## 2. 六维度定义

| 维度 | 内容 | 刻度/取值 |
|---|---|---|
| 配色 palette | bg/surface/border/text×3/accent×2/语义色，light+dark | 见 style-db/palettes.md |
| 布局 layout | 密度(紧凑12/舒适16/宽松24px gap)、排列(列表/网格/便当)、圆角档(0/4/8/14/20/999)、分隔线(发丝/粗线/无) | 见 style-db/layout-ui.md |
| UI | 边框(发丝1px/粗线3px/无)、阴影(无/柔/拟物内外/硬偏移/辉光)、状态(hover/active 位移/反色/发光)、按钮形状(直角/小圆/胶囊/斜切) | 见 style-db/layout-ui.md |
| 字体 font | 字族栈(系统安全栈;本地酒馆可webfont;MMD降级)、字号刻度、字重层级、数字等宽 | 见 style-db/fonts.md |
| 整体性 cohesion | 四维协调规则 + 搭配禁忌 | 本文件第4节 |
| 装饰 decoration | 光晕/扫描线/颗粒/辉光/漏光/硬阴影/无 + MMD约束 | 见 style-db/decoration.md |

## 3. 选择与分装协议

做美化前用 AskUserQuestion 先选基调组（素雅/柔和/科技/考究/玩味/氛围）再选具体风格（可给 2-3 个 preview 描述）；选定后默认整套 bundle（6 维度协调一致）应用；进阶可混搭——例如"配色用极简墨白、布局用便当格、装饰要颗粒"，AI 从各维度库取条目组装，混搭后必须跑第4节整体性检查。

## 4. 整体性守护

- 明暗一致：bg 与 text 必须满足对比 ≥4.5:1；换了 bg 必须同步 text。
- 装饰契合：霓虹/扫描线只配暗底；纸纹/颗粒只配亮哑光底；blur 光晕配深色或鲜艳底。
- 圆角一致：同一产出里状态栏与全局的 radius 取同档（除非用户明确要求单独调）。
- 搭配禁忌表：

| 冲突组合 | 为何冲突 |
|---|---|
| 0圆角野兽派 + 大圆角黏土柔影 | 0圆角传达硬朗/破坏感，大圆角传达圆润/亲和感，两者视觉语言对立，同屏造成风格撕裂 |
| 霓虹辉光 + 纸纹颗粒 | 霓虹需要深色背景衬托发光感，纸纹/颗粒是亮底哑光质感，底色逻辑互斥，两者叠加互相抵消 |
| 硬阴影 + 毛玻璃模糊 | 硬阴影强调实体轮廓与清晰边界，毛玻璃强调透明层叠与边界模糊，空间深度逻辑相反，叠用产生视觉矛盾 |

## 5. 项目级覆盖协议

1. 预设=基线，非锁死。选定风格后，用户对单 token 或单维度的任何改动（如"主色换 #C0392B"、"按钮改胶囊"、"状态栏圆角小一点、全局不变"），AI 直接在该项目产出（正则/CSS）里覆盖对应变量/规则。
2. 改动只落项目，不回写库：覆盖写进这张卡的产出 + 项目 `工作/美化决策.md`；`style-db/` 是只读基线，保持干净。
3. 整体性守护：改单点时检查是否破坏整体（对比度、明暗、圆角一致）；冲突则提示并给协调建议，而非闷头改。
4. 记录：每次覆盖在 `美化决策.md` 记一行（token、原值→新值、原因），断点续作可还原。
