# 角色卡创作规则

## 写作格式
- YAML中文格式、2空格缩进、xml标签包裹条目（如<character>、<personality>）
- 数据库格式优先：列表和键值对，不用散文段落
- 全文简体中文（桜→樱、宮→宫等日文汉字必须转换）
- 禁止占位符："某城市""某组织"等含糊表述禁止；不确定就问用户，不捏造

## 绝对零度原则
- 主观评价和判断 → 删除
- 陈旧/劣质比喻 → 改直接描述
- 堆砌形容词 → 精简（"红色的门"保留，"精美绝伦的门"精简）
- 代词和意象词模糊 → 改具体本意

## 八股化检查（发现即删/改写）
模糊词（似乎/仿佛/宛如/好像）、陈旧比喻（像小兽/心湖泛起涟漪）、八股微表情（嘴角微微上扬/眼中闪过一丝）、语气声线描写（带着xx的口吻）、极端情绪词（万念俱灰）、"不是...而是..."句式、大段心理描写、性格标签化（"她很温柔"）。
注意：正常动作词（笑了/皱眉/攥拳）不算八股，不矫正过度。

## 具体性检查
抽象→具体行为；笼统→具体细节；标签→行为展现。
错误："性格温柔善良" → 正确："遇到受伤的小动物会带回家照顾"

## 角色字段写法
- 外貌只写特征：只写偏离AI默认认知的部分（遮住名字能否认出是谁）
- 背景只写关键事件：删掉这条角色会变吗？不会→删
- 关系写具体画面：不写"深厚的感情"，写"有记忆起就在一起"
- 性格独立成条/块：基本信息是"她是谁"，性格是"她怎么行为"，混写会导致AI误调用

## chara_card_v3 字段分工
- description：角色核心设定（YAML+xml结构）
- personality：留空或一句话（详细性格进世界书条目）
- scenario：当前情境一句话
- first_mes：开场白（见 opening.md）
- 大体量设定 → 卡内世界书（character_book），不堆 description

## 自查清单
- [ ] 无主观评价（绝对零度）
- [ ] 无模糊词/陈旧比喻/八股微表情
- [ ] 无抽象标签（具体性）
- [ ] 全文简体中文
- [ ] 无占位符
