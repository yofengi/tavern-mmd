# 世界书设计

## 工作流程（三阶段）
阶段一 规划：判断卡型 → 列条目规划表 → 用户确认后才动工
阶段二 总纲：写世界观总纲条目（300-500字自然段压缩：世界类型/时代背景/核心规则/势力格局）
阶段三 展开：按规划表顺序逐条写（速览→角色→性格→物品/场景/事件→NPC）

## 卡型判断与蓝绿灯策略
| 类型 | 判定 | 策略 |
|---|---|---|
| 单角色卡 | 1个核心角色（拆分条目不算多角色） | 该角色所有条目蓝灯常驻 |
| 多角色卡 | ≥2核心角色 | 角色速览蓝灯 + 各角色详情绿灯（keys=全名,昵称,外号） |

## 条目规划表模板（给用户确认用）
| order | 条目 | position | 蓝/绿灯 | keys | 说明 |
|---|---|---|---|---|---|
| 1 | 世界观总纲 | 0 ↑Char | 蓝 | - | 始终存在 |
| 4 | 角色速览 | 0 ↑Char | 蓝 | - | 多角色卡必写 |
| 10+ | 角色设定（每角色一条） | 1 ↓Char | 视卡型 | 角色名 | <character>结构 |
| 35+ | 性格（每角色一条） | 1 ↓Char | 视卡型 | 同上 | <personality>独立 |
| 50-98 | 物品/能力/场景/事件 | 1 ↓Char | 绿 | 名称关键词 | |
| 100 | NPC | 1 ↓Char | 绿 | NPC名 | |
| - | 文风指导 | 2 ↑AT（MMD平台用蓝灯↓Char） | 蓝 | - | 见style.md |

## 配置规则
- 所有条目默认 preventRecursion + excludeRecursion
- 禁用@D depth≥1（打断对话流）；depth=0仅用于行为纠正指令（role=system），不放设定
- 绿灯keys：英文逗号分隔，含全名/昵称/外号
- token预算：单条目≤800字为宜；总纲≤500字；蓝灯总量控制（常驻全算token）——多角色卡蓝灯条目数≤5

## 内容写法
条目正文全部遵守 creation/character.md 的写作规则（绝对零度/八股化/具体性/简体中文/无占位符）。
世界观压缩四问：删了这句AI会错吗？是信息还是装饰？列表能替代吗？不看原文能理解吗？

## 平台差异
- 本地酒馆：全字段可用（含position 2/4/5/6、sticky等）
- MMD：保守只用 position 0/1 + 蓝绿灯 + 递归控制；文风条目改↓Char蓝灯（@AT支持未验证）
- 独立世界书json与卡内character_book字段结构不同——输出时见 output/worldbook-json.md
