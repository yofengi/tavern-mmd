# 角色状态与局势面板 生成协议（模型侧 · 蓝灯常驻条目）

每次回复正文结束后，**另起一行**用单个 `<g3>...</g3>` 块输出当前状态全量快照。**固态字段每轮必出**（即使没变也照原值重出）；**情境字段仅在对应情境那轮输出**。

> 由雷达法转换而来：取消散落正文的 `[键=值]` 信标与信标转换器，改单块结构化输出。tooltip 说明（效果/成因等）**内联进对应字段**，不再散落正文。

## 输出格式
```
<g3>
name={{user}};gender=男性;loc=星陨大陆-星陨王国-废弃法师塔-底层大厅;time=星历1472年9月15日 黄昏;armor=15;hp=45/45;mp=65/100;sp=80/100;race=3级精灵|100/300;cls=4级奥术师|200/400;stats=体质:10,力量:8,敏捷:14,智力:18,感知:15,魅力:16;equip=头饰:月夜辉冠|感知+3智力+6,主武器:紫木法杖|每击1d4钝+1d4奥术,全身:月白奥法袍;skill=法术:奥术飞弹|消耗5魔法发3飞弹,法术:霜冻新星|消耗15魔法冻结并2d6寒霜,天赋:黑暗视觉;goal=消灭石像鬼并寻找上层实验室的隐藏阶梯;event=1/3|暂无;turn=5
;bag=消耗品|强效恢复药剂2瓶/初级解毒剂/照明卷轴3卷
;opt=施放霜冻新星|消耗魔法制造大范围冰冻，延缓石像鬼B的侧翼包抄
</g3>
```

- 字段间用 `;` 分隔，键值用 `=`。
- **multi 字段**每条目重复一次该键（如 `opt=...;opt=...`）。
- 值里**禁止** `< > = ;` 与换行。各字段内部分隔符见下表。

## 分隔符约定

| 层级 | 符号 | 用于 |
|---|---|---|
| 字段间 | `;` | 所有字段 |
| 键值 | `=` | 所有字段 |
| 主分割 | `\|` | bar成因/level经验/kvlist名↔说明/enemy各段/opt简介↔内容/bag类↔物品/lines各条 |
| 列表项 | `,` | stats各属性/kvlist各条 |
| 次级项 | `/`或`、` | bag同类物品/enemy属性组 |
| 标签值 | `:` | stats`属性:值`/kvlist`槽位:名`/enemy属性`AC:14` |

## 铁律

- **推荐标准键名**（表第一列）；引擎对别名归一化容错，但标准键名最稳。
- **固态字段每轮必出**（面板无跨气泡状态、靠每轮全量，缺了引擎才向历史折叠兜底）。
- **情境字段 `enemy`/`opt`：只在该情境那轮输出，情境结束立即停止**（不每轮重复、不在情境外保留——面板随之消失，是防高刺激信息污染上下文的代谢机制，不是 bug）。
- **`bag` 每轮全量重出**（multi 字段不继承，靠模型每轮吐全集）。
- 选项：3~4 个差异化走向；可触发性爱在简介前加 `♥`；简介≤10字、内容约70~80字含动作言语。
- 只吐数据块，不解释、不加排版标记。
